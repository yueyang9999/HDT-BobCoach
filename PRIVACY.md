# Bob Coach 隐私说明

> 本说明适用于本仓库 GitHub Releases 提供的 Bob Coach 安装包。源码、CI 产物和第三方附件不属于官方安装包。

## 概述

Bob Coach 在本机处理酒馆战棋对局状态，不要求账号，也不主动收集 BattleTag、密码、支付信息、设备标识符或广告标识符。它不自动上传游戏日志、回放、用户档案或诊断文件。

## 本地读取与写入

插件可能读取 HDT 内存中的可见对局状态、`Power.log`/`Power_old.log`、HDT `config.xml`、Hearthstone `log.config`，以及自身的数据目录。它可在 `%APPDATA%\bob-coach\` 写入本地配置、诊断、对局相关快照和回放；这些内容可能包含对局模式、时间、可见对手状态或本机路径，应按个人数据处理。当前版本不会读取早期版本可能留下的外部饰品统计缓存。

`log.config` 默认只检查不写入。只有用户在 HDT 内的“Bob 教练”按钮查看目标路径、完整拟议变更并明确确认后，插件才可写入；文件变化冲突、无权限或拒绝确认时失败关闭。卸载不会自动删除共享 `log.config`。

## 网络行为

插件的核心本地基线不依赖 Bob Coach 自有服务。当前公开版不请求、不缓存、不展示 Firestone/Zero to Heroes 饰品统计，也不为该统计驱动 HearthstoneJSON 请求。首发不显示饰品报价选择提示；该显示开关只控制渲染，不会关闭已装备效果处理。

玩家装备饰品后，插件从 HDT 已知的本机对局状态读取 `GameState.ActiveTrinkets`（精确 `CardId` 列表），按版本化本地 `CardId` 规则计算确定性效果，并将结果用于后续行动、阵容和战斗决策。当前 `hdt-1.53.5-hearthdb-2026-07-22-r3` 规则事实版本覆盖 16 个精确饰品 ID，依据 HDT 1.53.5 / HearthDb build 245258 的维护者快照审计；未知 ID 只记录本地诊断并保守忽略，绝不按本地化名称或模糊文本推断效果。Oilcan (`BG30_MagicItem_705`) 的升本费用减免在行动合法性、资源计算和模拟中生效，费用缺失时使用本地 fallback 并钳制到 0，HDT 已提供的非负实时费用不再重复折扣。需要战团或手牌的战斗效果只读取对应玩家在 HDT 对局状态中已知的本机内存数据，不发送或持久化这些状态。这项处理不使用外部饰品统计或历史缓存。自动化规则测试只使用合成状态。

当前版本不读取、不迁移、不删除既有历史缓存。任何未来外部饰品统计适配器都必须先更新本说明和 `DATA_SOURCES.md`，重新设计并单独审批；不得上传 `Power.log`、回放、账号、对手、用户档案或设备标识。

## 删除与反馈

关闭 HDT 后，可运行包内 `UNINSTALL.ps1` 删除插件 DLL。只有显式指定 `-RemoveUserData` 才会删除默认 `%APPDATA%\bob-coach\` 数据目录；备份 DLL、共享 `log.config` 和外部 `BOB_COACH_DATA_ROOT` 不会被自动删除。

提交 Issue 前，请删除账号信息、密钥、完整日志、完整回放和绝对个人路径。新的网络端点、上传字段、存储路径、保留规则或账号功能必须先更新本文件。
