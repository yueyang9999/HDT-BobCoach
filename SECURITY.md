# Security Policy

## Supported version

Only the newest published Bob Coach beta is eligible for security fixes. Source snapshots and CI artifacts are not supported releases.

## Reporting a vulnerability

Use the repository's GitHub Security tab and select **Report a vulnerability** when private vulnerability reporting is enabled. Do not open a public Issue for a vulnerability that exposes code execution, path traversal, arbitrary file deletion, credential disclosure, package tampering, or private gameplay data.

If private reporting is not available, do not post exploit details or sensitive data publicly. Open a minimal Issue stating that a private reporting channel is required, without including the vulnerability details.

Include the affected version, Windows/HDT versions, impact, minimal reproduction, and whether the issue is reachable from the offline installer, plugin runtime, or local data files. Never include real account credentials, full logs, or unredacted personal paths.

No response or remediation SLA is promised. Maintainers will validate the report, coordinate a fix where feasible, and publish details only after users have a reasonable update path.
