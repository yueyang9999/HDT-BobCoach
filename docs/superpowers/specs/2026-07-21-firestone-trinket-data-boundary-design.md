# Firestone 饰品数据边界调整设计

日期：2026-07-21

## 目标

在不删除饰品识别、推荐规则和 UI 代码的前提下，停止公开版 Bob Coach 获取、缓存或使用 Firestone 饰品统计数据，保留可复用的数据校验能力，并尽量缩小改动范围以支持快速上线。同时把“报价阶段比较饰品”和“装备后理解饰品效果”拆成两个完全独立的能力：首发不展示饰品选择建议，但已装备饰品必须继续影响后续决策。

## 决策

公开版继续保留 HDT 饰品候选识别、HearthDb 卡牌事实、本地规则评估、饰品面板和渲染代码。首发版本隐藏饰品推荐显示，但不删除相关 UI 和规则实现。

生产运行时不得访问 Firestone/Zero to Heroes 饰品统计端点，不得缓存其选择率、平均名次、MMR 分层统计或将其用于评分、排序和展示。联网代码不能仅注释保留，因为注释代码仍会造成维护歧义和误恢复风险；应移除具体端点、生产初始化与调用入口。

`TrinketRecommendationsVisible = false` 只控制报价推荐的最终渲染，不得禁用 `ActiveTrinkets` 识别、本地效果解析或决策引擎集成。报价推荐服务与已装备效果服务不得共享启停条件。

## 数据流

允许保留的数据流：

```text
HDT / Power.log -> 饰品候选和局内状态
HearthDb         -> 卡牌 ID、名称、文本、类型和标签
Bob Coach        -> 本地规则评估（首发隐藏 UI 输出）
ActiveTrinketIds -> 版本化本地饰品规则 -> 后续行动、阵容和战斗决策
```

禁止的数据流：

```text
Firestone / Zero to Heroes -> 下载 -> 缓存 -> 校验 -> 评分或 UI
```

## 组件调整

### 生产插件

- 保留饰品选择生命周期、规则评估、推荐服务、面板和渲染代码。
- 通过单一、可测试的显示开关隐藏首发版本的饰品推荐 UI。
- 移除 `BobCoachPlugin` 对 Firestone 饰品统计更新器的创建、Build 通知和释放调用。
- 移除生产源码中的 Firestone 饰品统计 URL，确保发行 DLL 不包含该端点。

### 已装备饰品效果

已装备效果采用来源无关的本地边界：

```text
ActiveTrinketIds
    -> TrinketEffectRegistry
    -> ActiveTrinketContext
        -> EffectiveGameRules
        -> FeatureExtractor
        -> ActionScoring
        -> CombatSimulator
```

- 已知饰品只按精确 `CardId` 匹配版本化本地规则，不根据本地化名称或模糊文本修改合法性、费用和资源。
- 硬规则在行动枚举和资源计算前合并进 `EffectiveGameRules`；不能仅在最终分数上补权重。
- 卡牌与阵容价值修正进入特征和行动评分；战斗开始、召唤和成长类效果进入战斗模型。
- 未识别饰品保守忽略未知效果并限频记录诊断，不改变任何规则或评分。
- 首批只注册当前模型能够精确表达且能由合成状态验证的规则，不声称覆盖全部饰品。
- 所有饰品规则测试只使用合成状态和本地固定 ID，不需要 Firestone 或其他统计数据。

### 校验链路

- 保留模型完整性、Build、卡牌 ID、时间、哈希、重复项和数值范围校验能力。
- 校验器改为来源无关，不硬编码 `source=firestone` 或 Firestone 的时间周期。
- 自动测试只使用合成的最小数据，不下载、不提交、不复刻 Firestone 实际统计。
- 为缩小改动面，现有统计模型、通用 HTTP 获取器、存储器和校验器文件保留；生产插件没有调用它们的运行路径。
- `TrinketStatsUpdater` 改为无网络、无定时器、无缓存读写的禁用协调器，状态明确为“没有已授权外部来源”，并保留未来接入已授权适配器的接口边界。
- 删除 Firestone 具体端点、专用 JSON 解析和自动重试入口；不读取、不迁移、不删除用户机器上已经存在的历史缓存。

### 文档与声明

- README、`DATA_SOURCES.md`、`PRIVACY.md`、`NOTICE`、架构和依赖文档统一说明：公开版不请求、不缓存、不展示 Firestone 饰品统计。
- Firestone 仅作为历史评估背景被客观说明，不把其名称、数据或服务描述为 Bob Coach 的当前数据源。
- 后续如开发 Firestone/BepInEx 模组或取得明确授权，必须重新设计并单独审批数据适配器。

## 错误处理

移除 Firestone 运行时入口后，网络失败不应影响插件加载、局内识别、购买提示或发现提示。饰品推荐显示关闭时，不弹出错误、不启动重试定时器、不创建饰品统计缓存。

## 验证标准

- 饰品 UI、规则和识别源码仍存在并可编译。
- 默认运行路径不显示饰品推荐。
- 默认隐藏开关不影响已装备饰品上下文解析和效果计算。
- 已知已装备饰品能通过精确 `CardId` 修改对应硬规则、卡牌协同或战斗状态；未知 ID 仅产生诊断。
- 报价推荐服务和已装备效果服务没有共享启停条件或统计数据依赖。
- 生产插件不创建 Firestone 更新器，也不发起相关请求。
- 仓库生产源码和发行 DLL 不包含 `static.zerotoheroes.com` 或 Firestone 饰品统计 URL。
- `%APPDATA%\bob-coach` 不再新增 Firestone 饰品统计缓存。
- 合成数据校验测试继续覆盖接受、隔离、Build 不匹配、重复 ID、未知 ID 和范围错误。
- `npm test`、仓库验证脚本、Release 构建和 `git diff --check` 通过。
- 上线前仅做针对本次变更的快速冒烟检查，不重跑已经完成且与数据源调整无关的完整 VM 生命周期验收。

## 不在本次范围

- 不接入 HSReplay Tier7 或其他聚合统计服务。
- 不删除饰品 UI、规则评估或 HDT/HearthDb 事实读取代码。
- 不用 Firestone 数据补全本地饰品规则，也不承诺首批覆盖全部饰品。
- 不新增外部数据源。
- 不发布 Release、不修改 CI/CD、不处理既有用户数据。
