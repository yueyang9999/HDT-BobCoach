# BobCoach 安装教程与功能展示页设计

日期：2026-07-24

## 目标

让第一次安装 HDT 插件的炉石玩家，仅凭一页中文图文教程完成安装；同时用五张真实对局截图直观说明 BobCoach 能在酒馆战棋中提供哪些建议。

## 安装教程

安装主流程调整为：

1. 完全退出 HDT。
2. 重新打开 HDT，进入“选项 > 插件”。
3. 点击“打开插件文件夹”。
4. 将解压后的 `BobCoach.dll` 和 `BobCoach.dll.sha256` 一起复制到插件目录。
5. 返回 HDT，在插件列表中开启 BobCoach。

教程中的 HDT 设置页以本地原图 `屏幕截图 2026-07-24 151703.png` 为唯一来源。成品图裁取左侧插件设置区域，保留 BobCoach 版本、启用开关和“打开插件文件夹”按钮，去掉右侧广告、统计和其他无关界面。原图不进入 Git。

现有“运行窗口打开目录”的图片和说明由新的 HDT 内置入口替代。复制文件步骤继续使用现有隐私清理后的文件夹截图。Markdown 与独立 HTML 使用相同的本地图片和相同的操作顺序；即使图片无法显示，正文也必须足以完成安装。

## 功能展示页

新增中文 Markdown 和可离线打开的独立 HTML 页面，按玩家一局游戏的常见决策顺序展示：

1. 购买建议：标出当前商店中更值得购买的随从。
2. 升本建议：提示当前回合是否适合升级酒馆等级。
3. 技能提示：提醒更合适的英雄技能使用时机。
4. 饰品推荐：在可选饰品中给出推荐。
5. 发现选择：在三选一等发现界面中辅助比较。

每项只包含功能名、面向玩家的一句话说明、对应实战截图。页面显著说明“BobCoach 只提供建议，不会替玩家点击或操作游戏”。功能截图来自用户提供的五张原图，最终提交版本只遮挡左下方玩家名称和左侧分数，并压缩到最长边不超过 1600 像素；其余游戏画面保留，原图不进入 Git。

## 页面与导航

- `docs/user/INSTALL.md` 和 `docs/user/INSTALL.html`：同步安装流程与新截图。
- `docs/user/FEATURES.md` 和 `docs/user/FEATURES.html`：新增功能展示页。
- `docs/user/images/install/`：只存放安装教程成品图。
- `docs/user/images/features/`：只存放裁剪、脱敏、压缩后的功能展示成品图。
- `README.md` 和 `docs/README.md`：增加功能展示入口，不在首页堆放五张大图。

独立 HTML 不引用 CDN、字体服务或远程图片，确保下载后可直接用浏览器离线打开。页面正文宽度采用适合长文阅读的约 960 像素上限，移动端保持单列且图片不横向溢出。

## 公开内容规则

先更新 `AGENTS.md`，允许 `docs/user/images/features/` 中经过裁剪、脱敏和压缩的最终功能截图。其余原始截图、验证截图和用户数据截图继续禁止提交。`NOTICE` 补充说明截图中的 Hearthstone、HDT 和其他第三方画面及标识归各自权利人所有。

## 验收标准

- 安装教程准确展示“打开插件文件夹”和 BobCoach 开关所在位置。
- Markdown 与 HTML 的安装步骤、文件名和版本事实一致。
- 五项功能均有名称、简短说明和对应图片。
- 所有提交图片均已脱敏，单张最长边不超过 1600 像素，仓库内不存在原始大图。
- 所有本地图片引用存在，HTML 离线打开不依赖网络。
- 仓库规定的测试、Release 构建、内容扫描和 `git diff --check` 全部通过。
