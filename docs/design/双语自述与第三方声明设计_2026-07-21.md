# 双语自述与第三方声明设计

## 目标

让中文用户默认看到可直接理解的项目介绍，同时为英文用户提供内容等价的入口；在 README 中清晰指向安装、隐私、数据来源、第三方声明、支持与 MIT 许可，避免把未使用的数据源或第三方背书写成事实。

## 仓库边界

- 只修改当前公开仓库 `yueyang9999/HDT-BobCoach`。
- 旧私有归档仓库 `yueyang9999/HDT-BObcoash` 保持只读，不同步新文档。
- `README.md` 保持中文，作为 GitHub 默认首页。
- 新增 `README.en.md`，两个文件顶部提供 `中文 | English` 相互切换链接。

## 内容结构

中英文 README 保持相同的信息层级：产品定位、公开测试版本、系统要求、安装、隐私与联网、数据来源与第三方声明、构建与测试、支持与自愿赞赏、免责声明、MIT 许可。

英文版以中文 README 的已验证事实为准，不新增未经验证的兼容性、功能或发布承诺。中文 README 只补语言入口和更明确的文档导航，不重写已验收的安装步骤。

## 中文下载与安装入口

- `README.md` 继续作为 GitHub 默认首页，面向中文用户；只有 `README.en.md` 作为英文入口。
- 中文首页在项目介绍之后直接提供“下载与安装”板块，不要求普通用户先理解 Release、Asset 或源码压缩包等 GitHub 术语。
- Windows 11 64 位和 Windows 10 64 位使用同一个 `BobCoach-0.2.0-beta.1-win-x64.zip`。不复制或重命名为两个内容相同的系统专用安装包，避免制造错误的二进制差异和重复校验文件。
- 系统映射必须同时写明验证等级：Windows 11 24H2 x64 已完成实机验收；Windows 10 22H2 x64 仅为技术兼容，尚未完成专用实机验收。不得将二者都写成“已验证可用”。
- 首页使用“中文安装教程（新手从这里开始）”作为 `docs/user/INSTALL.md` 的链接文字。保留文件路径以维持现有链接稳定，不要求用户通过英文文件名判断用途。
- Release 下载板块按 Windows 11 和 Windows 10 分行指向同一个 ZIP，并把 `.zip.sha256` 单独标为校验文件。紧邻下载入口说明 GitHub 自动生成的 `Source code (zip)` 和 `Source code (tar.gz)` 不是安装包。
- 英文 README 保持相同的包映射和兼容性事实，但默认浏览路径、主操作文案和安装教程优先服务中文用户。

本次不改变安装脚本、插件 DLL、包内容、版本号或发布工件哈希；只改善仓库首页、中文安装导航和现有 Release 的说明。未来只有在 Win10 与 Win11 确实需要不同二进制时，才拆分系统专用安装包。

## 数据来源与第三方边界

- Firestone/Zero to Heroes 仅作为历史评估背景；当前生产运行时不请求、不缓存、不使用 Firestone/Zero to Heroes 饰品统计，也不读取、迁移或删除早期版本可能留下的历史缓存。
- 当前生产运行时只声明代码实际使用的版本化本地 `CardId` 规则，以及 HearthstoneJSON/HearthSim 游戏事实的受限校验边界；未知饰品 ID 失败关闭。
- HSReplay 当前未被代码接入、抓取、打包或再分发；文档明确写出这一点，避免让用户误以为 Bob Coach 使用 HSReplay 数据。
- Firestone、HSReplay、HearthSim、HDT、Blizzard 等名称仅用于说明兼容性、数据来源或未使用边界，不表示关联、赞助或背书。
- 根目录 MIT 许可只覆盖 Bob Coach 原创代码及有权许可的原创材料，不重新授权第三方软件、数据、统计、商标或游戏内容。

## 支持与自愿赞赏

`SUPPORT.md` 继续作为“为爱充电”政策页。所有功能永久免费，赞赏不解锁功能、不影响推荐、不形成付费支持等级，也不出现在 Hearthstone 或 HDT overlay 内。

在没有真实、可撤销的外部赞赏链接前，不创建 `.github/FUNDING.yml`，不放占位链接，也不提交个人收款二维码。后续优先接入独立赞赏平台页面；如最终选择二维码，应先确认收款主体、隐私展示、替换方式、退款与平台规则，再单独评审。

## 验证

- 检查中英文 README 内部链接均指向仓库现有文件。
- 检查两种语言对版本、系统兼容、安装路径、联网边界和许可的表述一致。
- 检查 Windows 10 与 Windows 11 的下载入口都指向同一个官方 ZIP，且验证状态没有混写。
- 检查中文首页和 Release 页面都能直接识别中文安装教程，并明确排除 GitHub 自动生成的源码压缩包。
- 搜索确认没有把 HSReplay 写成当前数据来源。
- 运行仓库文档/合同测试、`validate_repository.ps1` 和 `git diff --check`。
